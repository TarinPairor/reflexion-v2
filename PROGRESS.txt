WHAT IS REFLEXION?
------------------
A physical dementia screening device. A Raspberry Pi Zero 2W with a mic and
speaker conducts a voice-based cognitive assessment with an AI assistant called
Aria. Results are automatically scored and displayed on a web dashboard.
Designed for elderly patients — no screen, no touch, voice only.

The assessment has 4 stages:
  1. Free conversation (~2-3 min) — warm up, captures natural speech for ML
  2. TICS-m questions — date, city, digits forward/backward, word introduction
  3. More free conversation (~3-4 min) — delay period, more speech for ML
  4. Delayed word recall + goodbye

After the session, patient speech is analysed by an ML model to predict
Healthy vs MCI (Mild Cognitive Impairment). Results + TICS scores are
displayed on a web dashboard.


HARDWARE
--------
- Raspberry Pi Zero 2W (headless, no screen)
- Waveshare USB audio module (mic + speaker, 44.1kHz mono)
- Speaker is ~10cm from mic (causes echo — known issue)
- Connected to Mac via phone hotspot


SOFTWARE
--------
On Mac:
  - node server.js        → web server at localhost:3000 (dashboard + API)
  - python audio_server.py → Flask ML server at localhost:5001

On Pi:
  - pi_audio_bridge.py    → connects to OpenAI Realtime API via WebSocket,
                            streams mic audio, plays AI voice through speaker,
                            records patient speech, sends WAV to Mac for ML

ML Model:
  - Random Forest classifier
  - 9 acoustic features from eGeMAPSv02 (openSMILE)
  - Trained on 255 samples (public Alzheimer's dataset)
  - Accuracy: 65%. Better at detecting MCI (recall 0.78) than healthy (0.50)
  - Trained by CS groupmate Tarin. Repo: github.com/TarinPairor/tigerlaunch


HOW TO RUN A SESSION
--------------------
On Mac (first time only):
  Double-click setup.command — installs all dependencies and sets up .env

On Mac (every session):
  1. Switch Mac to phone hotspot
  2. Double-click start.command — starts both servers and opens dashboard

On Pi (every session):
  3. If pi_audio_bridge.py was updated:
       scp /Users/justinwong/tigerlaunch/pi_audio_bridge.py pi@reflexion.local:~/pi_audio_bridge.py
  4. ssh pi@reflexion.local
  5. source ~/reflexion-env/bin/activate && python ~/pi_audio_bridge.py
  6. Let Aria finish all 3 stages — wait for "All done." in terminal
  7. View results: http://localhost:3000/dashboard → Live Sessions tab

Pi .env file (at ~/.env on Pi):
  OPENAI_API_KEY=sk-...
  MAC_IP=172.20.10.4     ← check with: ipconfig getifaddr en0
  AUDIO_DEVICE=USB PnP


WHAT IS WORKING
---------------
- Full 4-stage voice assessment runs autonomously ✓
- Patient speech recorded (AI voice excluded) ✓
- WAV sent to Flask ML server → MCI prediction returned ✓
- TICS-m scores auto-generated by AI via function calling ✓
- Results saved to session_results.json (all sessions kept as history) ✓
- Dashboard Live Sessions tab shows all sessions, expandable ✓
- Colour-coded results: green/yellow/red for TICS score, green/red for ML ✓
- AI voice: shimmer ✓
- Mic fully muted while Aria speaks — unmutes only after buffer fully drains ✓
- Watchdog: detects AI silence >30s and sends nudge to continue ✓
- Patient silence detection: 15s silence → Aria checks in, another 15s → session ends ✓
- Patient explicit end: patient can say "I need to go" → Aria confirms → ends session ✓
- Partial sessions: early-end sessions saved with NIL for unanswered questions ✓
- Dashboard: incomplete sessions flagged with ⚠, NIL shown for missing scores ✓
- setup.command: one-click installer for new Macs ✓
- start.command: one-click launcher for both servers ✓
- Scoring: strict on date (±1 day leniency on day only) and city (Singapore) ✓
- Scoring: lenient on digits (transcription errors allowed) ✓
- Scoring: word recall only counted from Stage 4, not Stage 2 repetition ✓
- Debug log: session_debug.json saved after each session (previous kept as session_debug_prev.json) ✓


KNOWN ISSUES
------------
- Echo: mostly fixed — mic muted while buffer has audio. Occasional tail-end pickup remains
- Transcription accuracy: Whisper occasionally mishears short/accented answers
- Random noise: USB audio module occasionally picks up RF interference,
  transcribed as gibberish — hardware limitation, no software fix
- Language: occasionally detects wrong language. NOT restricting to English
  since Singapore is multilingual (English, Mandarin, Malay, Tamil)
